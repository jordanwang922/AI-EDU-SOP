# AI 亲子教育 SOP · 移动端 H5 设计风格说明（design.md）

> 版本：**1.1**  
> 定位：**明亮小清新、透气、偏糖果 pastel**，面向以妈妈为主的家长群体；**避免莫兰迪式灰雾感**（不显脏、不显闷）。  
> 本文与 Figma 文件 `AI亲子教育SOP · H5设计规范 v1` 中的 Variables 同步；开发以本文 Hex + Figma Token 为准。

---

## 1. 风格关键词（对内对齐）

| 关键词 | 含义 |
| --- | --- |
| **明亮小清新** | **高明度背景**（薄荷浴白 / 浅青绿氛围）+ **饱和适中**的主色（薄荷青绿 Teal）+ **亮色点缀**（玫粉 CTA），整体「清爽有朝气」，不是灰蒙蒙 |
| **妈妈向但不幼龄** | CTA 可用 **明亮粉红**，但主品牌仍用 **清新的 Teal**，避免满屏「橡皮粉」廉价感 |
| **可信教育产品** | 文字用 **偏 Teal 的深中性色**或 **Slate**，对比清晰；不用灰褐莫兰迪字色 |
| **刻意规避** | **莫兰迪式低饱和灰绿 / 灰褐底 / 灰粉**：易显旧、显闷；本项目 **v1.1 起废止** v1.0 燕麦灰褐系 |

---

## 2. 色彩趋势参考（为何这样配）

以下仅为**灵感来源**；**执行以色板表格为准**。

1. **高明度 pastel / 日系小清新 UI**  
   - 背景用 **Teal-50 / Green-50** 系浴白，带一点点色相，比「奶茶灰底」更干净、更「亮」。  
2. **时尚 pastel 组合（偏 Pantone Powdered Pastels 但提亮）**  
   - **Mint / Aqua** 作主色传递「成长、清爽」；**Pink** 作唯一强操作色，形成清晰层次。  
3. **母婴产品共识（迭代理解）**  
   - 仍可柔和，但 **饱和度略抬升**，避免灰浊色让父母觉得「界面旧旧的」。

## 3. 品牌色板（执行规范 · v1.1 明亮版）

### 3.1 背景与表面

| Token | Hex | 说明 |
| --- | --- | --- |
| `color/bg/page` | `#F0FDFA` | **薄荷浴白（Teal-50）**：整体明亮、带轻微青绿氛围，取代灰燕麦底 |
| `color/bg/elevated` | `#FFFFFF` | **纯白卡片**，提升通透感 |
| `color/bg/subtle` | `#CCFBF1` | **浅薄荷分区**，用于二级背景 / chip，仍保持清新 |

### 3.2 文字（清爽对比，拒绝灰褐）

| Token | Hex | 说明 |
| --- | --- | --- |
| `color/text/primary` | `#134E4A` | **深 Teal 灰**：标题 / 正文（比灰褐 `#3D38..` 更干净） |
| `color/text/secondary` | `#64748B` | **Slate**：次级说明（标准中性灰，不偏褐） |
| `color/text/tertiary` | `#94A3B8` | 辅助 / Caption |

### 3.3 品牌与操作（明亮 Teal + 明亮玫粉）

| Token | Hex | 说明 |
| --- | --- | --- |
| `color/brand/primary` | `#14B8A6` | **薄荷青绿 Teal-500**：主按钮、链接、选中 —— **比鼠尾草饱和更高、更「亮」** |
| `color/brand/primary-pressed` | `#0D9488` | Teal-600 按下态 |
| `color/action/cta` | `#EC4899` | **明亮玫粉 Pink-500**：复制提示词等强操作（的妈妈向辨识度） |
| `color/action/cta-pressed` | `#DB2777` | Pink-600 |

> 标签点缀可选用 **`#38BDF8` Sky-400**（小天蓝）或 **`#A78BFA` Violet-400**（浅紫），**小面积**即可。

### 3.4 边框与语义色（清透，少用灰褐）

| Token | Hex | 说明 |
| --- | --- | --- |
| `color/border/subtle` | `#99F6E4` | Teal-200 **清透薄荷边**，取代灰褐边框 |
| `color/semantic/success` | `#34D399` | Emerald-400 **明亮成功绿** |
| `color/semantic/warning` | `#FBBF24` | Amber-400 |
| `color/semantic/error` | `#F87171` | Red-400 |

### 3.5 版本差异（v1.0 → v1.1）

| v1.0（偏莫兰迪） | v1.1（明亮小清新） |
| --- | --- |
| 燕麦 `#FAF7F4`、灰褐字、灰褐边 | 薄荷浴白 `#F0FDFA`、Teal 字色、**Teal-200 清透边** |
| Sage `#8FAE9D`、雾粉 `#E5989B` | **Teal-500 `#14B8A6`**、**Pink-500 `#EC4899`** |

## 4. 布局与组件气质

- **圆角**：卡片 / 按钮用大圆角（与 Figma `radius/md`、`radius/lg` 一致）。  
- **留白**：偏大间距；**白卡片浮在浅薄荷底上**，层次更清晰。  
- **图标**：线性或浅色填充，可用 **略饱和** 的 Teal / Pink，避免灰浊图标。  

---

## 5. 无障碍与实现注意

- Teal / Pink 按钮若使用 **白字**，请在实现侧测 **WCAG AA**；不足则改为 **浅底 + 深色字** 的「浅色按钮」变体。  
- 开发使用 CSS 变量与 Figma Variables **同名映射**。
