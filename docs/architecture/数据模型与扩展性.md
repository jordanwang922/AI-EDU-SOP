# 数据模型与扩展性

> 版本：0.3  
> 数据库：**PostgreSQL 16**（开发与生产统一）  
> ORM：**Prisma 6.9**，schema 位于 `web/prisma/schema.prisma`

## 1. 当前数据模型（已落地）

### 1.1 ER 概览

```
AdminUser
Category ──< Problem ──< ProblemTag >── Tag
Feedback
TemplateField
```

### 1.2 表结构详情

| 模型 | 字段 | 说明 |
| --- | --- | --- |
| **Category** | `id`, `name`, `description`, `sortOrder`, `isActive`, `createdAt`, `updatedAt` | 19 个一级类目，`sortOrder` 控制排序 |
| **Problem** | `id`, `title`, `categoryId`, `sortOrder`, `essence`, `mistakes`, `principles`, `steps`, `script`, `prompt`, `isPublished`, `createdAt`, `updatedAt` | 394 条问题，内容字段：问题本质/常见误区/正确原则/操作步骤/沟通话术/AI提示词 |
| **Tag** | `id`, `name` (UNIQUE), `createdAt`, `updatedAt` | 1686 个搜索关键词标签 |
| **ProblemTag** | `problemId`, `tagId` (复合主键) | 问题-标签多对多关联，3531 条 |
| **AdminUser** | `id`, `email` (UNIQUE), `passwordHash`, `createdAt`, `updatedAt` | 后台管理员账号 |
| **TemplateField** | `id`, `name` (UNIQUE), `sortOrder`, `createdAt`, `updatedAt` | 内容模板字段定义 |
| **Feedback** | `id`, `source`, `type`, `content`, `rating`, `status`, `createdAt` | 用户反馈 |

### 1.3 关系

- `Category` 1:N `Problem`（级联删除）
- `Problem` N:M `Tag`（通过 `ProblemTag`，级联删除）

## 4. 搜索机制

当前搜索基于 Tag 模型：
- 用户输入关键词 → 匹配 `Tag.name`（`contains` 模糊搜索）→ 通过 `ProblemTag` 关联找到对应问题
- 标签包含同义词（如"说谎/撒谎"）、年龄段标签（"小学/初中/高中"）、关联概念

远期可扩展：
- PostgreSQL `pg_trgm` 扩展实现模糊匹配
- `tsvector` 全文检索
- `pgvector` 向量检索 + RAG

## 5. 扩展预留

| 扩展方向 | 方案 |
| --- | --- |
| 用户登录 | 新增 `EndUser` 模型（手机号/微信 OpenID） |
| 收藏/历史 | 新增 `UserFavorite` / `ViewHistory` 关联表 |
| 热度排序 | Problem/Category 增加 `clickScore` 字段或独立统计表 |
| 向量检索 | Problem 增加 `embedding` 字段（`pgvector`） |
| 付费/订单 | 新增 `Order` / `Entitlement` 模型 |
