# 系统设计规则（信息架构 · 排序 · 与 PRD 差异说明）

> 版本：**1.0**  
> 本文档沉淀**已共识的工程与产品规则**；与 `docs/prd/…V1.1.md` 冲突时，以**本文件 + `产品设计说明书` 中显式约定**为准（PRD 保留历史问题树与字段全集）。

## 1. 家长端浏览层级（减少点击）

### 1.1 目标

在保持「一级大类」心智的前提下，**取消单独的问题列表层**，使家长从进入大类到看到**完整解决方案页**的点击次数 **≤ 2**（不含首页进入大类的一次点击）。

### 1.2 约定结构（相对 PRD 原文的调整）

| PRD 历史结构（四层） | 现行规则（三层） |
| --- | --- |
| 一级大类 → 二级主题页 → 具体问题列表 → 词条详情 | **一级大类 → 词条列表（叶子）→ 词条详情** |

- **一级大类**：与 PRD 一级目录一致（如「学习问题」「亲子沟通」），首页入口不变。
- **第二层屏幕**：进入某一级大类后，**直接展示该大类下所有已发布词条（叶子）**，每一条对应过去的「具体问题」，点击进入 **同一套 PRD 词条详情模板**（本质、误区、原则、SOP、提示词、话术等一次呈现）。
- **取消的中间层**：不再强制经过「二级主题汇总页 → 三级问题列表页」两次点击。PRD 中的「二级分类」（如「拖拉磨蹭」「不爱学习」）在后台仍可保留为 **分组、筛选维度或标签**，家长端可用 **分组标题 / 折叠区块** 在同一列表页内展示，但**不作为单独路由页面层级**（除非后续 A/B 另有决策）。

### 1.3 与后台内容模型的关系

- 词条内容仍服从 PRD **词条字段**与发布状态；仅 **前台导航路径**收束。
- 后台维护分类树时，可将原「二级」节点标记为 **仅用于分组 / 排序 / 统计，默认不在 H5 生成独立列表页**（由实现时在配置或字段上体现，如 `nav_depth` / `show_as_hub`）。

## 2. 排序规则（手工 + 点击热度自动上浮）

### 2.1 目标

在冷启动与运营可控的前提下，**点击越多的入口越靠前**，减少大多数用户的滑动与二次查找。

### 2.2 适用范围

| 对象 | 是否适用热度排序 |
| --- | --- |
| **一级大类**（首页或大入口列表） | 是 |
| **词条在一级下列表中的顺序**（第二层屏幕内的行） | 是 |
| **后台内部分组节点**（若存在二级分类节点） | 可选；若前台同一列表内混合展示，可对「分组内条目」单独计权 |

### 2.3 混合策略（推荐）

1. **基础顺序**：运营配置的 **`sort_order`（手工排序）** 作为默认静态顺序。
2. **行为信号**：匿名会话下采集 **`category_impression` / `category_click`、`content_click`**（命名示意），聚合为 **滚动窗口内的点击计数或加权分**（可按日/周衰减，避免老热点霸榜）。
3. **合成排序**：  
   `effective_rank = f(manual_priority, click_score, pinned)`  
   - **Pinned / 置顶**：手工可将某些一级大类或词条 **锁定前 N 位**，不参与热度挤出或仅参与弱挤出。  
   - **冷启动**：无数据时完全遵循 `sort_order`。  
4. **透明度**：后台提供「当前生效排序预览」与「热度指标」列，便于运营纠偏。

### 2.4 合规与隐私

- 无登录场景下以 **匿名会话 / 设备指纹（若启用）** 聚合即可；需在隐私政策中说明「用于改进内容排序」。
- 禁止用热度排序暴露敏感个体信息；仅存聚合统计。

## 3. 架构与数据落地要点

### 3.1 API 行为（示意）

- **GET 一级入口列表**：返回顺序已按「系统设计规则 §2」合成后的列表。
- **GET 某一级大类下的词条列表**：同上；可选查询参数 `group_by=subcategory` 仅在同一响应内分组，不增加路由层级。
- **POST 详情曝光 / 点击**：幂等或节流写入，用于热度累计（具体事件名与批量上报策略见实现阶段 ADR）。

### 3.2 数据表扩展（与 `数据模型与扩展性.md` 配套）

- `categories`：保留 `sort_order`；增加 **`click_score` / `pinned` / `sort_mode`**（示意）或独立 **`category_stats`** 按周期滚动。
- `contents`：保留 `sort_order`；增加 **`click_score`** 或与词条绑定的统计表。
- **`behavior_events`**（可选）：原始点击流，离线聚合为分数。
